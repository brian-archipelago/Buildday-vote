How the AI builds the poll

Two different models, a rolling 5-minute transcript window, and a merge step that has to be additive — or pitches disappear. Here's the whole pipeline in four diagrams.

1End-to-end flow

Mic on the room → transcription model → store → analyzer model → every screen.

transcription model analyzer model infra / data flow
Two different OpenAI models do two different jobs. gpt-4o-transcribe handles speech→text in real time; gpt-5.4-mini periodically reads the rolling transcript and decides where one pitch ends and the next begins. The store is the only source of truth — every screen subscribes via SSE.

2One analyzer pass — the prompt

What the analyzer model actually sees, and what it returns.

The analyzer is a boundary detector, not a transcriber. It looks at the timestamped transcript and decides "the stretch from t=14s to t=42s is one pitch about X." Each entry it returns is either an existing pitch (echoed by id, possibly with refined fields) or a new pitch it just identified.

3The merge — additive, never destructive

How the model's response combines with what's already in the store. (This is the part that broke before.)

Why the production bug happened: the old merge did s.pitches = whatever-the-model-returned. When 15 pitches had accumulated but the rolling 5-minute transcript only mentioned the last one or two, the model "consolidated" and returned 5. The merge replaced 15 with 5 — pitches gone. And the periodic timer kept firing for ~5 minutes after Stop, running the merge three more times with progressively thinner context. 15 → 12 → 5.
The fix: the merge is now strictly additive. The model can only refine unlocked pitches in place or propose brand-new ones. Removal is admin-only (Delete button + tombstone in _deletedTitles). Plus: the periodic timer skips when recording is off and nothing new has arrived since the last pass — no more ghost passes thrashing the model.

4Timing — when the analyzer fires

Two triggers, two cadence rules, and one safety brake after Stop.

Each green dot is a transcript segment landing in the rolling window. Each blue tick is an analyzer pass running. The pre-fix bug was on the right side of this diagram: timer ticks kept firing after Stop, eating the still- growing-thin context and dropping pitches. Now they stop when there's nothing new to chew on.

The whole mental model in one sentence

Transcription is one model, pitch-boundary detection is another, and the analyzer's job is to be a careful editor of an existing list — not the sole author of it. Locked pitches (admin-edited) and now all unlocked pitches are protected from being dropped; the worst the analyzer can do is propose a bad new entry, which the admin's Delete button (with its tombstone) cleans up. And if anything goes sideways, the snapshot ring buffer means Undo gets you back.
